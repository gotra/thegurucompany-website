How to compile and install GIT on RHEL

January 1, 2014 by Rajeev

Today I was trying to install the latest version of Git on my workstation provided by the client. The workstation had an older version of Git and I needed Git version 1.9 or above to work with intelliJ IDEA version 13.1

Git can be installed either from source or from the console, however its recommended to install from the source if you want the latest.

To install Git from source code use the following procedure

  1. Install the libraries Git depends on: curl, zlib, openssl, expat, and libiconv 
    # yum install curl-devel expat-devel gettext-devel \
                        openssl-devel zlib-devel gcc
  2. Download the latest snapshot from the Git web site, located here:http://git-scm.com/downloads
  3. Compile and install 
                            # tar -zxf git-1.9.3.tar.gz
# cd git-1.9.3
# make prefix=/usr/local
# make prefix=/usr/local install
  4. Test the installation of Git 
    # git -- version
    It should display you the version of Git.

To install Git from console is quiet simple

# yum install git
