Mapping to a view in Spring MVC without controller

February 1, 2014 by Rajeev

If you are writing a spring controller to just return a view without any business logic STOP NOW!

Since spring 3.0 you can use the following tag in your XML configuration

<mvc:view-controller path="/" view-name="index"/>

In case you prefer to use java config file instead of XML use the code below

                    
@Override
public void addViewControllers(ViewControllerRegistry registry) {
    registry.addViewController("/").setViewName("index");
}
According Spring documentation: This tag is a shorcut for defining a ParameterizableViewController that immediately forwards to a view when invoked. Use it in static cases when there is no Java Controller logic to execute before the view generates the response.
